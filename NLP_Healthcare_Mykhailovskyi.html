Problem Statement

Central Question

How do encoder-only (BioClinicalBERT) and decoder-based (MedGemma-4B) architectures compare when fine-tuned on two core medical NLP tasks — NER and QA — under a constrained compute budget?

Why It Matters

  • Clinical NLP enables automated chart review, coding, and decision support
  • Domain-specific models outperform general LLMs but trade-offs are unclear
  • LoRA/QLoRA makes billion-parameter models accessible on a single GPU

Two Tasks

Task 1 — Named Entity Recognition (NER)
Label each token in biomedical text with a UMLS semantic type (e.g., T038 = Biologic Function, T103 = Chemical)
Task 2 — Medical Question Answering
Select the correct answer (A–D) from USMLE-style clinical questions requiring multi-step medical reasoning

Scope Constraints

  • Single GPU (Google Colab Pro A100)
  • PEFT only (LoRA / QLoRA) for the 4B model
  • Public, reproducible datasets only

Key Engineering Decisions

  • Subword tokens → -100 label (ignored in loss)
  • Exponential class weights: exp(−nC/N)×100, O-class = 0.5
  • Custom WeightedTrainer injects per-class loss
  • Cosine LR schedule with 10% warmup, LR = 6e-5
  • Early stopping (patience = 2 on macro F1)
  • Published checkpoints: alexd063/bio-clinicalbert-finetuned-medmentions

Task 1 — NER: MedGemma-4B with QLoRA

Generative NER Approach

Unlike token classification, MedGemma generates a free-text sequence:
aspirin: B-T103, prescribed: O, daily: O, …
Each token is explicitly labelled in the output text.

Training Config

  • QLoRA: 4-bit NF4, rank 16, alpha 16, no dropout
  • Unsloth for 2× training speed
  • SFTTrainer, max 300 steps (≈ 20% of 1 epoch)
  • LR = 2e-4, batch = 4, grad accum = 4 (eff. 16)
  • Evaluation: qualitative inspection of 3 val samples

Sample Prompt Format

### Instruction:
Extract medical entities and label each
token with its NER tag.

### Input:
Patient presents with acute appendicitis
and elevated CRP levels.

### Response:
Patient: O, presents: O, with: O,
acute: B-T047, appendicitis: I-T047, …
Only 300 steps trained (underfitting). No quantitative F1 metric — evaluation is visual only. Parsing "token: tag" format from free text is fragile.

Why BioClinicalBERT Underperforms

  • Max sequence length 128 tokens — USMLE questions often exceed this
  • Truncation discards part of the question context
  • Bidirectional encoder not designed for open-ended reasoning
  • Effectively near-random (26% ≈ 25% baseline)
BioClinicalBERT is architecturally ill-suited for multi-step clinical reasoning. The result confirms that task–architecture mismatch is real.

MedGemma-4B Performance

  • 55.7% on test (1,273 samples) — 2× random baseline
  • Official MedQA benchmark: 64.4% (without fine-tuning!)
  • Fine-tuning degraded performance vs. pre-trained baseline
  • Possible causes: only 2 epochs, LR too high, minimal warmup
Fine-tuning on a small dataset can hurt a strong pre-trained model — catastrophic forgetting risk. The pre-trained MedGemma already knows medicine; instruction fine-tuning should be conservative.

Comparative Analysis: Encoder vs. Decoder

Dimension Bio_ClinicalBERT (Encoder) MedGemma-4B (Decoder)
NER Macro F1 0.65 (full test, token classifier) 0.27 (subset, generative)
QA Accuracy ~26% (≈ random) 55.7%
GPU time / task ~20–40 min (110M params) ~1.5–2.5h (4B params, QLoRA)
Trainable params 110M (full fine-tune) 33M (0.76% via LoRA)
Inference speed Fast (single forward pass) Slow (autoregressive decoding)
Best for Classification, token labeling, NER Reasoning, generation, QA
Key takeaway: Use the right architecture for the task. Encoders excel at structured extraction (NER). Decoders excel at open-ended reasoning (QA). Neither model is universally better — the comparison validates task–architecture alignment as the dominant factor in performance.

Discussion: What Worked & What Didn't

What Worked

  • Class weighting significantly improved NER on frequent entities
  • QLoRA made 4B-param model trainable on a single GPU
  • MedGemma outperformed BERT 2:1 on QA (55.7% vs 26%)
  • Custom WeightedTrainer was clean and reusable
  • Cosine LR schedule + early stopping stabilized training
  • ECE metric added calibration insight beyond accuracy
  • All models published to HuggingFace Hub for reproducibility
  • Unsloth gave 2× training speedup at no cost

What Didn't Work

  • CRF layer: fails on post_init() bug (1-line fix missed)
  • Synthetic data augmentation: only 33 samples generated; no measurable impact
  • BERT QA: near-random accuracy — wrong architecture for reasoning
  • MedGemma NER: only 300 steps, no quantitative metrics
  • Fine-tuning MedGemma QA degraded vs pre-trained baseline (64.4% → 55.7%)
  • Comparative NER evaluation: asymmetric test set sizes
  • No random seeds in most BERT notebooks — not fully reproducible
  • Summarization task dropped (originally proposed)
CSCI E-222 · NLP for Healthcare

Limitations

Data & Evaluation

  • MedMentions uses zero-shot label protocol — test entities unseen during training, artificially suppressing metrics
  • MedGemma NER evaluated only qualitatively on 3 samples — no reliable F1
  • BERT QA truncates to 128 tokens — full context lost for longer questions
  • MedQA is US-centric (USMLE) — does not represent global clinical practices
  • MedMentions-MTI881-NER dataset has no license — legal status unclear

Model & Compute

  • Single GPU (A100) limits experiment scale and ablation studies
  • No hyperparameter tuning — all settings are single-run
  • MedGemma NER severely under-trained (300 of ~1,462 steps)
  • No confidence intervals or multi-seed averaging
  • Catastrophic forgetting risk: fine-tuning may degrade pre-trained medical knowledge in MedGemma
  • Bio_ClinicalBERT trained on MIMIC-III only — US ICU notes, limited generalization

Responsible Use & Ethical Considerations

Clinical Safety

These models must NOT be used for clinical decisions. All outputs require review by a qualified medical professional.
  • Models trained on limited datasets — real patients have far more complex presentations
  • MedGemma can hallucinate plausible but incorrect clinical facts with high confidence
  • NER errors (false positives/negatives) could misclassify diagnoses or medications
  • BioClinicalBERT was pre-trained on US ICU data — may be biased toward US patient populations

Data & Licensing

  • MedMentions-MTI881-NER: no license stated — usage is legally ambiguous
  • MedQA-USMLE: CC-BY-4.0 — permissive, attribution required
  • MedGemma: Google Health AI Developer Foundations terms — must be accepted; restricts commercial use
  • MIMIC-III (pre-training data for BioClinicalBERT): PhysioNet credentialed access — not distributable
All published HuggingFace checkpoints (alexd063/*) should carry explicit license notices and responsible-use disclaimers.
